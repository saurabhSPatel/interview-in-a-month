Core Patterns

P01 Two Pointers

EasyUse when searching ordered pairs or narrowing from both ends.

  • Trigger: sorted array and pair/triple constraints.
  • Time: O(n) after sorting.
  • Prompt: find pair with target sum, max area, 3-number zero sum.

Why Use This Pattern

Two pointers allow efficient scanning of arrays with constraints on pairs or subarrays, avoiding brute-force O(n²) checks by moving pointers based on conditions.

Implementation Sketch

def two_sum(nums, target):
    nums.sort()
    left, right = 0, len(nums) - 1
    while left < right:
        current_sum = nums[left] + nums[right]
        if current_sum == target:
            return [left, right]
        elif current_sum < target:
            left += 1
        else:
            right -= 1
    return []

Example Problem

Problem: Given a sorted array, find two numbers that sum to a target.
Solution: Use two pointers from start and end, move based on sum comparison.

Common Pitfalls

  • Forget to sort if array isn't sorted.
  • Off-by-one errors in pointer movement.

P02 Sliding Window

MediumUse for contiguous subarray/substring constraints.

  • Trigger: longest/shortest/maximum under condition.
  • Time: usually O(n).
  • Prompt: minimum window, at-most-k distinct, max sum length-k.

P03 Prefix Sum

EasyTurn many range queries into O(1) lookups.

  • Trigger: repeated sum in [l..r].
  • Time: build O(n), query O(1).
  • Prompt: subarray sum equals k, equilibrium index, range updates.

P04 Binary Search

EasySearch over sorted data or monotonic predicate.

  • Trigger: first/last occurrence, lower/upper bound.
  • Time: O(log n).
  • Prompt: rotated array search, insertion point, peak element.

P05 Binary Search on Answer

MediumSearch the feasible value space, not array index.

  • Trigger: minimum capacity/time/rate with monotonic checker.
  • Time: O(n log range).
  • Prompt: ship packages, eating speed, split array largest sum.

P06 Hashing + Frequency Maps

EasyCount occurrences and detect duplicates/constraints fast.

  • Trigger: frequency, distinct count, anagrams.
  • Time: O(n), space O(n).
  • Prompt: top-k frequent, valid anagram, happy number variants.

P07 Fast/Slow Pointers

MediumFind cycles/midpoint without extra memory.

  • Trigger: linked-list cycle or runner strategy.
  • Time: O(n), space O(1).
  • Prompt: cycle start, middle node, duplicate number mapping.

P08 Stack Basics

EasyLIFO for matching, undoing, and expression processing.

  • Trigger: parentheses matching, expression eval.
  • Time: O(n).
  • Prompt: valid parentheses, simplify path, decode string.

P09 Monotonic Stack

MediumTrack nearest greater/smaller efficiently.

  • Trigger: nearest boundary and contribution counting.
  • Time: O(n) amortized.
  • Prompt: histogram area, daily temperatures, subarray minimum sum.

P10 Deque / Monotonic Queue

MediumMaintain window min/max in O(1) amortized updates.

  • Trigger: sliding window extrema or shortest feasible window with prefix sums.
  • Prompt: sliding max, shortest subarray with sum >= k.

P11 Heap / Priority Queue

MediumPick next best candidate repeatedly.

  • Trigger: top-k, merge sorted streams, shortest path frontier.
  • Prompt: kth largest, merge k lists, task scheduler.

P12 Intervals + Sorting

MediumSort by start/end then merge/select.

  • Trigger: overlap, coverage, schedule conflict.
  • Prompt: merge intervals, insert interval, non-overlap erase.

P13 Sweep Line

HardTransform interval problem into event processing.

  • Trigger: active set over time/position.
  • Prompt: skyline, calendar overlap count, resource timeline.

P14 Difference Array

MediumDo many range updates cheaply.

  • Trigger: add value to many [l..r] updates.
  • Prompt: car pooling capacity, range addition, flight booking counters.

P15 Linked List Operations

MediumPointer choreography with sentinel nodes.

  • Trigger: reverse, merge, reorder, partition.
  • Prompt: reverse k-group, merge two sorted lists, LRU primitives.

P16 Recursion + Backtracking

MediumEnumerate choices and prune invalid branches.

  • Trigger: combinations, permutations, board-search.
  • Prompt: N-Queens, subset sum, word search.

P17 Tree DFS

MediumPost-order for DP, pre-order for path generation.

  • Trigger: subtree result aggregation.
  • Prompt: max path sum, diameter, lowest common ancestor.

P18 Tree BFS

EasyLevel-wise traversal for shortest edges and layer features.

  • Trigger: level order, nearest target by hops.
  • Prompt: right-side view, min depth, zigzag order.

P19 Trie

MediumPrefix search and incremental dictionary operations.

  • Trigger: autocomplete, prefix match, word dictionary.
  • Prompt: insert/search prefix, wildcard word search, suggestion engine.

P20 Bitmask + Bit Tricks

MediumCompact state representation and fast toggles.

  • Trigger: subsets, parity, small-state DP.
  • Prompt: subset generation, single number, maximum xor.

Advanced Patterns

P21 Union-Find

MediumDynamic connectivity with path compression and union by size.

  • Trigger: merging groups and checking connectedness quickly.
  • Prompt: redundant edge, network connectivity operations, account merging.

P22 Graph DFS/BFS

MediumTraversal over adjacency lists for reachability and components.

  • Trigger: islands/components/path existence.
  • Prompt: number of islands, clone graph, bipartite check.

P23 Topological Sort

MediumDAG ordering via indegree or DFS states.

  • Trigger: prerequisite dependency resolution.
  • Prompt: course schedule, alien dictionary, safe states.

P24 Shortest Path

HardDijkstra for non-negative edges, Bellman-Ford for negative edge allowances.

  • Trigger: minimum cost/path under weighted edges.
  • Prompt: network delay, cheapest flights with constraints, min effort path.

P25 Minimum Spanning Tree

HardKruskal/Prim for minimum total connection cost.

  • Trigger: connect all nodes with least total weight.
  • Prompt: connect cities cheaply, min cost to connect points.

P26 Greedy + Proofs

MediumPick local best only when exchange argument holds.

  • Trigger: interval selection, task ordering, minimum operations.
  • Prompt: jump game, partition labels, gas station loop.

P27 DP 1D/2D

MediumOverlapping subproblems + optimal substructure.

  • Trigger: count ways, min cost, longest relation in sequence/grid.
  • Prompt: coin change, LCS, edit distance, house robber.

P28 State-Machine DP

HardModel finite states and transitions over steps/time.

  • Trigger: stock with cooldown/fee/transactions, process lifecycle constraints.
  • Prompt: buy/sell stock variants, holding/not-holding transitions.

P29 Interval DP + Memo Search

HardOptimize over subranges with split-point transitions.

  • Trigger: "choose k split in range i..j".
  • Prompt: burst balloons, strange printer, matrix chain-style optimization.

P30 Segment Tree / Fenwick Tree

HardSupport range queries and updates over an array with logarithmic performance.

  • Trigger: multiple range sum/min/max queries with updates.
  • Prompt: range sum query, range min query, dynamic order statistics.

P31 Rolling Hash / String Fingerprint

HardCompare substrings quickly and detect repeated patterns without brute-force checks.

  • Trigger: substring matching, palindrome checks, duplicate subarrays.
  • Prompt: string search, repeated substring, longest duplicate substring.

P32 State Graph Search

HardModel complex problem states as graph nodes and search with BFS/DFS/A* over the state space.

  • Trigger: puzzles, shortest sequence of transformations, dynamic move systems.
  • Prompt: sliding puzzle, word ladder, minimum operations to reach target state.

Machine Coding Add-on

Core Prompt Families

  • Rate limiter (token bucket, sliding window)
  • Thread-safe cache with TTL and eviction
  • Worker pool with backpressure and graceful shutdown
  • Pub-sub event bus and retry queue
  • Leaderboard with top-k + rank window
  • Mini key-value store with expiration

Evaluation Rubric

  • API clarity and extensibility
  • Concurrency safety and lock strategy
  • Failure handling and idempotency
  • Testability and separation of concerns
  • Complexity awareness and bottleneck identification
Machine Coding Flow 1) Clarify API contracts and constraints 2) Define structs/interfaces and invariants 3) Implement happy path 4) Add concurrency safety + failure behavior 5) Add test hooks and extension points
