01Key Concepts

Raft Consensus

  • Leader election: term-based, candidate wins majority votes
  • Log replication: leader → followers, committed when majority ACK
  • Safety: committed entry survives any leader change
  • Election timeout: random 150–300ms to avoid split vote
  • Used by: etcd (Kubernetes), CockroachDB, Consul
  • Your Hyperledger Fabric uses PBFT variant (BFT Raft)

Vector Clocks + Lamport

  • Lamport timestamp: logical counter, inc on send/receive
  • Establishes happens-before but not concurrent events
  • Vector clock: per-process counter array
  • VC detects concurrency: neither dominates = concurrent
  • Used for conflict detection in Dynamo, CRDTs

Bloom Filter

  • Probabilistic set membership: no false negatives, yes false positives
  • k hash functions → k bits set in bit array
  • Space efficient: 10 bits/element for 1% FPR
  • Used by: Cassandra (skip SSTable reads), Chrome (safe browsing), Kafka (log dedup)
  • Counting BF: deletions allowed (increment/decrement)

HyperLogLog

  • Approximate cardinality (count distinct) in O(1) space
  • 1.5KB memory estimates billions of unique items
  • Algorithm: leading zeros of hashed values
  • Standard error: ~1.04 / sqrt(m) where m = registers
  • Redis PFADD/PFCOUNT uses HLL — your Redis expertise

TCP/IP Stack

  • 3-way handshake: SYN → SYN-ACK → ACK (1.5 RTT)
  • Congestion control: AIMD, slow start, cubic
  • HOL blocking: stream blocked by lost packet
  • TIME_WAIT: 2×MSL (2 min) after close — port reuse issue
  • SO_REUSEPORT: multiple processes bind same port (kernel LB)

HTTP/2 vs HTTP/3 vs QUIC

  • HTTP/2: multiplexing over single TCP, header compression (HPACK)
  • HTTP/2 HOL: TCP packet loss blocks all streams
  • HTTP/3: QUIC (UDP-based), per-stream loss recovery
  • QUIC: 0-RTT reconnect (session resumption), connection migration
  • Your Gin/HTTP services: HTTP/2 with TLS termination at LB

02Must-Know Deep Dives

🔥 TLS 1.3 Handshake — What Happens on Every HTTPS Request

// TLS 1.3 (1-RTT, vs TLS 1.2's 2-RTT) Client → Server: ClientHello (supported cipher suites, key_share) Server → Client: ServerHello (chosen cipher, key_share, Certificate, Finished) // Encryption starts here — much earlier than TLS 1.2 Client → Server: Finished (verify server cert, compute master secret) Client → Server: HTTP Request (already encrypted) // 0-RTT Session Resumption: Client → Server: Early Data (pre_shared_key from previous session) // Replays possible — only safe for idempotent requests (GET, HEAD)

Certificate validation: OCSP stapling (server includes cert revocation proof), CT logs (certificate transparency — detect rogue CAs). For your CBDC security background: mutual TLS (mTLS) is Hyperledger Fabric's channel authentication mechanism.

🔥 Encoding: Protobuf vs JSON vs Avro

JSON: human-readable, schema-optional, 100% overhead. Type coercion issues. Use for APIs, config, debugging.

Protobuf: binary, schema required (proto files), 5-10x smaller than JSON, backward compatible with field numbers. Required fields are required — don't remove them. Use for internal service communication where schema evolution is controlled.

Avro: schema in header, schema evolution via schema registry. Kafka standard for streaming. Your Kafka events should use Avro + Confluent Schema Registry for schema evolution guarantees.

// Amortized Complexity — why ArrayList append is O(1) amortized // Double array when full: 1→2→4→8→16→32... operations = 2n total copies // For n appends: n + n/2 + n/4 + ... = 2n work = O(1) amortized per append // Go slices: same doubling strategy. append() = O(1) amortized, O(n) worst case

04Quick Revision

CS FUNDAMENTALS FLASH CARDS

Raft vs Paxos?
Same safety guarantees, different framing. Raft: understandable, leader-based, strong leader. Paxos: foundational theory, flexible, harder to implement. etcd uses Raft.
Bloom filter FP rate?
FPR ≈ (1 - e^(-kn/m))^k where k=hash funcs, n=items, m=bits. 10 bits/item, 7 hash funcs → ~1% FPR. Never false negatives.
HTTP/2 HOL blocking?
TCP packet loss blocks ALL HTTP/2 streams (one TCP stream). HTTP/3/QUIC: per-stream loss recovery — losing packet only blocks that stream.
Vector clock concurrent?
VC A concurrent with B if neither A ≤ B nor B ≤ A (neither dominates). Need CRDT or user-visible conflict to resolve.
Protobuf backward compat?
Add fields: OK (old code ignores unknown fields). Remove required fields: NEVER (breaks old decoders). Change field types: NEVER. Rename: OK (uses field number, not name).
Amortized O(1) append?
Doubling strategy: total copies over n appends = 2n. Cost per append = 2n/n = O(2) = O(1) amortized. Worst case single append: O(n).
